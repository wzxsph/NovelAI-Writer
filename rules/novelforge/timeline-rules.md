# 时间线/事件排序规则

## 时间线基础

每章必须明确：
- 发生时间（第X章/第X天/时间点描述）
- 事件顺序
- 时间跨度

## 事件记录

每个事件记录：
```json
{
  "event_id": "evt_001",
  "chapter": 1,
  "timestamp": "第一天/早晨",
  "description": "事件描述",
  "participants": ["char_001", "char_002"],
  "consequences": ["结果1", "结果2"]
}
```

## 时间顺序规则

1. **因果律**：原因必须在结果之前
2. **并发处理**：同一时间多事件 → 记录为同一时间戳
3. **时间跳跃**：跨章节时间跳跃需明确标注

## 时间线一致性

- 检测时间矛盾（同一角色同一时间出现在两处）
- 检测逻辑错误（结果先于原因）
- 检测遗漏事件（应有但未记录）

## 时间表示

| 方式 | 示例 |
|------|------|
| 章节编号 | 第1章 |
| 时间段 | 早晨/中午/夜晚 |
| 日期 | 第一天/第三天 |
| 时间点 | 辰时/午时/申时 |

## 一致性检查

时间线检查维度：
- temporal_sequence（事件顺序）
- 时间点冲突
- 时间跨度合理性
