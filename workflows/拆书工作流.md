# 拆书工作流

## 触发条件
用户执行 `/novelai import --path /path/to/novel` 或要求导入现有小说。

## 输入
- 小说文件路径（支持 .txt, .md 格式）
- 或小说文本内容（直接粘贴）

## 工作流程

### Step 1: 文本解析

**执行**: 解析输入文本

**处理**:
1. 识别章节标题（# 第X章 / Chapter X / "第X章"）
2. 分割章节内容
3. 统计字数和章节数

**输出**: 章节列表 [{title, content, word_count}]

### Step 2: 章节摘要

**执行**: 对每个章节生成摘要

参考 @prompts/大纲生成.txt（结构类似）

**对于每个章节**:
- 提取核心事件
- 提取出场角色
- 提取关键情节点

**输出**: 章节摘要列表

### Step 3: 结构分析

**执行**: 分析整体结构

**分析**:
1. **基础骨架**：故事主线、核心冲突
2. **人物脉络**：角色关系网络、角色弧线
3. **技法行动**：叙事风格、章节节奏

**输出**: 三份分析文档

### Step 4: 状态迁移

**执行**: 将分析结果写入 state_document.json

**映射**:
- 章节摘要 → chapters 数组
- 人物 → characters 数组
- 关系 → relationships 数组
- 物品 → items 数组
- 情节线 → plot_threads 数组

**输出**: 完整的 state_document.json

### Step 5: 章节归档

**执行**: 将原始章节保存

**结构**:
```
chapters/
├── 原始/
│   ├── chapter_001_raw.md
│   └── chapter_002_raw.md
```

### Step 6: 生成结构化大纲

**执行**: 基于分析生成结构化大纲

**输出**: outline.md（符合 novelai-writer 格式）

## 输出

1. `chapters/` 目录：原始章节文件
2. `state_document.json`：完整状态文档
3. `outline.md`：结构化大纲

## 约束

1. 原始文本必须保留
2. 状态文档必须可验证
3. 大纲格式必须符合 novelai-writer 标准
